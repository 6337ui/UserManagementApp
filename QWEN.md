# UserManagementApp - Информационный файл проекта

## Обзор проекта

UserManagementApp - это веб-приложение для управления пользователями (посетителями), разработанное как тестовое задание. Приложение предоставляет удобный интерфейс для просмотра, добавления, редактирования и поиска пользователей.

### Технологический стек

- **Vue 3** – прогрессивный фреймворк для построения пользовательских интерфейсов
- **Pinia** – официальное решение для управления состоянием в Vue 3
- **Tailwind CSS** – utility-first фреймворк для стилизации
- **Vite** – современная сборка для быстрой разработки

### Структура проекта

```
src/
├── components/             # Компоненты интерфейса
│   ├── atoms/              # Атомарные компоненты (кнопки, поля ввода и т.д.)
│   │   ├── Button.vue      # Универсальная кнопка
│   │   ├── Input.vue       # Поле ввода
│   │   ├── Select.vue      # Выпадающий список
│   │   ├── Label.vue       # Метка
│   │   ├── Container.vue   # Контейнер
│   │   ├── Table.vue       # Таблица
│   │   ├── Thead.vue       # Заголовок таблицы
│   │   ├── Tbody.vue       # Тело таблицы
│   │   ├── Tr.vue          # Строка таблицы
│   │   ├── Th.vue          # Ячейка заголовка
│   │   └── Td.vue          # Ячейка данных
│   ├── composables/        # Мелкие переиспользуемые компоненты
│   ├── header.vue          # Шапка приложения
│   ├── footer.vue          # Подвал приложения
│   └── table.vue           # Таблица пользователей
├── stores/                 # Хранилища состояния
│   └── visitorsStores.js   # Хранилище для управления данными пользователей
├── App.vue                 # Главный компонент приложения
├── main.js                 # Точка входа приложения
└── styles/                 # Стилевые файлы
    ├── tokens.css          # CSS-переменные для темизации
    └── components/         # Стили компонентов
```

## Сборка и запуск

### Установка зависимостей

```bash
npm install
```

### Запуск в режиме разработки

```bash
npm run dev
```

Приложение будет доступно по адресу `http://localhost:5173`.

### Сборка проекта в продакшен

```bash
npm run build
```

### Предварительный просмотр продакшен-сборки

```bash

```

## Архитектура и функциональность

### Хранилище (Store)

- Используется Pinia для управления состоянием
- Данные хранятся в localStorage
- Поддерживает добавление, обновление и удаление пользователей
- Реализованы фильтры (Присутствующие/Отсутствующие/Без фильтра)
- Поиск по имени пользователей

### Компоненты

#### Атомарные компоненты (atoms/)
- `Button` - универсальная кнопка с различными вариантами и размерами
- `Input` - поле ввода с различными вариантами и размерами
- `Select` - выпадающий список с различными вариантами и размерами
- `Label` - метка с различными размерами
- `Container` - контейнер с различными вариантами и размерами
- `Table`, `Thead`, `Tbody`, `Tr`, `Th`, `Td` - компоненты для создания таблиц

#### Основные компоненты
- `Header` - шапка приложения с логотипом, поиском и информацией о посетителях
- `Table` - таблица с данными пользователей
- `Footer` - панель фильтров
- `Popup` - модальное окно для добавления/редактирования пользователей
- Компоненты в папке `composables` - переиспользуемые элементы интерфейса

### Функции

- Просмотр списка пользователей
- Добавление новых пользователей
- Редактирование существующих пользователей
- Удаление пользователей
- Фильтрация по статусу присутствия
- Поиск по имени
- Сохранение данных в localStorage

## Особенности реализации

1. **Управление состоянием**:
   - Все данные пользователей хранятся в централизованном хранилище
   - Используется localStorage для сохранения данных между сессиями

2. **Интерфейс**:
   - Адаптивный дизайн с использованием Tailwind CSS
   - Модальные окна для добавления/редактирования
   - Интерактивные элементы управления

3. **Структура компонентов**:
   - Использование атомарного дизайна для создания переиспользуемых компонентов
   - Атомарные компоненты обеспечивают единообразие и гибкость стилизации
   - Использование CSS-переменных для темизации компонентов

4. **Темизация**:
   - Все цвета и стили определены через CSS-переменные в tokens.css
   - Атомарные компоненты используют эти переменные для обеспечения согласованности
   - Поддержка темного режима через медиа-запросы

## Разработка

### Стиль кодирования

- Использование Composition API в Vue
- Соглашения об именовании переменных и функций в соответствии с JavaScript/ES6+
- Использование стрелочных функций для краткости и читаемости
- Использование директив Vue и реактивности Pinia

### Атомарный дизайн

- При создании новых компонентов рекомендуется использовать существующие атомарные компоненты
- Для новых типов элементов интерфейса создавать новые атомарные компоненты
- Все стили должны использовать CSS-переменные из tokens.css

### Тестирование

- Проект не содержит явно видимых файлов тестов, но может использовать инструменты тестирования, указанные в документации Vue и Pinia

### Вклад в проект

- Для внесения изменений рекомендуется следовать установленным соглашениям об именовании и структуре проекта
- Использовать компонентный подход при добавлении новой функциональности
- Обеспечивать совместимость с localStorage для сохранения данных пользователей
- При добавлении новых визуальных элементов использовать атомарные компоненты и CSS-переменные
